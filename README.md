1.	Usecase
1.1	Login:
Aktor (dokter, kasir, dan farmasi) dapat melakukan login ke sistem
1.2	Melakukan Pembayaran:
Pasien dapat melakukan pembayaran
1.3	Melihat riwayat tansaksi:
Aktor (pasien dan kasir) dapaat melihat riwayat transaksi
1.4	Memesan dan menerima:
Pasien dapat menerima struk dan obat dari kasir dan pasien dapat memesan kamar pada kasir
1.5	Mengelola data pasien:
Kasir dapat mengelola data pasien
1.6	Mengelola kamar:
Kasir dapat mengelola kamar
1.7	Memberi Obat:
Kasir memberikan obat dari bagian farmasi ke pasien
1.8	Melihat data pasien:
Aktor (kasir, farmasi dan dokter) dapat melihat data pasien
1.9	Membuat laporan:
Kasir membuat laporan tentang data pasien dan laporan keuangan
1.10	Meracik Obat:
Farmasi dapat meracik obat sesuai anjuran dokter
1.11	Memeriksa pasien:
Dokter memeriksa pasien dan mendiagnosa penyakit pasien


2.	Diagram Activity
2.1	Pasien memulai pembayaran
2.2	Kasir memeriksa informasi pembayaran dan data pasien di sistem
2.3	Sistem menampilkan informasi pembayaran
2.4	Sistem akan menghitung tagihan pasien
2.5	Pasien membayar sesuai tagihan
2.6	Kasir menginput data pembayaran ke sistem
2.7	Sistem menyimpan data pembayaran pasien
2.8	Sistem mencetak laporan pemmbayaran
2.9	Kasir menerima laporan dari sistem
2.10	Kasir membuat rekap laporan
